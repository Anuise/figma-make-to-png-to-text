# Figma Make 專案分析

此領域將 Figma Make 匯出的專案轉換為可供工程規劃使用的畫面、工作流與工項資料。

## Language

**來源專案（Source Project）**:
由 Figma Make 匯出的完整 React 原始碼資料夾，是每次分析的唯讀輸入。分析期間不會將它轉換成其他前端框架，也不會直接修改其內容。
_Avoid_: Figma 檔案、設計稿

**來源版本（Source Revision）**:
來源專案在特定內容 fingerprint 下形成的不可變快照；同一分析作業可依序關聯多個來源版本而不覆寫歷史產物。
_Avoid_: 工作副本、分析版本

**工作副本（Working Copy）**:
分析作業從來源專案建立的私有副本，所有依賴安裝、暫時修改與執行活動都發生於此。
_Avoid_: 來源專案、備份

**實作模式（Implementation Mode）**:
工程規劃對來源專案的使用方式；預設為需求原型，只有明確選擇延續實作時才把既有程式碼視為正式系統基礎。
_Avoid_: 分析模式、執行模式

**目標技術設定（Target Technology Profile）**:
工程規劃所假設的正式前端、後端、資料庫、驗證、部署與測試技術條件；缺少時工項保持技術中立並降低估時信心。
_Avoid_: 來源專案技術棧、分析工具技術棧

**分析工具（Analysis Tool）**:
由單一使用者在本機執行，重複處理來源專案並產出畫面、工作流及工程工項資料的內部工具。
_Avoid_: 單次分析腳本、通用網站分析平台

**分析作業（Analysis Run）**:
針對一個來源專案執行的分階段分析；各階段彼此獨立，階段內可按資料量分批，產物均可保存並從有效狀態繼續處理。多個分析作業可以排隊，但同時只有一個作業執行。
_Avoid_: 單次執行、全流程重跑

**分析批次（Analysis Batch）**:
在單一分析階段內，由一組輸入資料形成的可獨立執行、驗證、保存及重試的處理單位。
_Avoid_: 一次全量處理、對話回合

**畫面（Screen）**:
在網站中可透過導覽與互動抵達，且在視覺呈現或使用者目標上具有明顯差異的 UI 狀態；不以 URL 作為唯一邊界。
_Avoid_: 頁面、路由、截圖

**候選畫面（Candidate Screen）**:
自動探索時發現、可能符合畫面定義，但仍待人工確認、合併或排除的 UI 狀態。
_Avoid_: 最終畫面、重複畫面

**完整畫面截圖（Full Screen Capture）**:
在 Desktop `1440 × 900` viewport 下，透過頁面捲動涵蓋正式畫面完整垂直內容的截圖；長度超過 viewport 不會被截斷。
_Avoid_: viewport 截圖、首屏截圖

**探索設定（Exploration Configuration）**:
用來讓分析工具抵達無法自動發現之畫面的版本化探索指示；有效版本保存於分析工具，YAML 僅供匯入與匯出。
_Avoid_: 測試腳本、網站設定、來源專案設定

**受保護畫面（Protected Screen）**:
需要登入、權限或特定初始化資料才能抵達的畫面；沒有對應探索設定時，狀態為未探索而非分析失敗。
_Avoid_: 私有頁面、登入頁

**工作流（Workflow）**:
以一個使用者目標為單位，由前置條件、操作步驟、預期結果及例外組成，並關聯其涉及的畫面。
_Avoid_: click 錄製、測試案例、DOM 操作序列

**工作流草稿（Workflow Draft）**:
由分析工具根據探索結果與證據產生、尚未經使用者確認的工作流版本；草稿可被修改、合併或排除。
_Avoid_: 正式工作流、已核准需求

**證據（Evidence）**:
支持畫面、工作流或工項結論的可追溯資料，例如完整畫面截圖、操作記錄、Playwright trace 或來源程式碼位置。
_Avoid_: 推測、備註

**穩定 ID（Stable ID）**:
在同一分析作業及其版本中識別畫面、工作流、工項、API 與證據的持久識別碼，用於跨產物追溯而不依賴顯示名稱。
_Avoid_: 列號、顯示名稱

**分析工作簿（Analysis Workbook）**:
由分析工具產出的 Excel 交付物，透過多個工作表與穩定 ID 整合畫面、工作流、工程工項、API 及證據。
_Avoid_: 報表、截圖清單

**人工確認（Human Review）**:
使用者在分析工具中確認、修改、合併或排除候選畫面、工作流草稿、工程工項、API 與工時估算的階段；確認結果才可成為下游分析或正式交付的輸入。
_Avoid_: Excel 審核、最終 QA

**交付匯出（Delivery Export）**:
預設只從最新且已確認的分析產物產生 ZIP 交付包，內容包含分析工作簿、原始證據與 manifest；交付包不是分析作業的主要資料來源。
_Avoid_: 匯入、主資料庫、單一 Excel 檔

**儲存模式（Storage Mode）**:
分析作業建立時選定的產物保存方式：本機檔案系統或 S3 相容物件儲存；同一分析作業執行期間不可切換。
_Avoid_: 暫存位置、下載位置

**已觀察 API（Observed API）**:
從來源程式碼、網路請求或執行證據中實際確認存在的 API 契約資訊。
_Avoid_: 推測 API、建議 API

**建議 API（Proposed API）**:
為完成正式工作流而推導出的 API 需求，必須標示推導依據與確認狀態，不代表來源專案已有實作。
_Avoid_: 已存在 API、確定契約

**資料模型需求（Data Model Requirement）**:
從工作流、畫面、表單、mock data 或程式碼觀察或推導出的邏輯實體、欄位、驗證與關係；不等同特定資料庫 schema。
_Avoid_: SQL schema、ORM model、ERD

**橫切工項（Cross-cutting Task）**:
跨越前端、後端或基礎設施邊界的工程工作，只有在存在證據或目標技術設定要求時才產生，並標示推導依據與確認狀態。
_Avoid_: 樣板工項、Frontend Task、Backend Task

**工程工項（Engineering Task）**:
可獨立驗收的前端、後端或橫切工程工作；一般以基準工程師 2–16 小時完成為目標，超過時必須拆分或標記待拆。
_Avoid_: 技術步驟、Epic、商業需求

**建議實作順序（Suggested Implementation Order）**:
依工程工項、API、資料模型與工作流的相依關係推導出的技術執行順序，不代表商業優先級。
_Avoid_: Business Priority、產品 Roadmap

**基準工程師（Baseline Engineer）**:
具有一年相關前端或後端經驗、熟悉工項採用的技術棧，但尚未熟悉該來源專案的工程師。
_Avoid_: 初級工程師、平均工程師

**工時估算（Effort Estimate）**:
基準工程師完成工項所需的小時數，包含理解程式碼、實作、必要測試、本機除錯、驗收驗證及一般 Code Review 修正，不含需求會議、等待、正式 QA／UAT、部署申請或不可預見的既有系統故障。估算由最可能工時、合理區間、信心及假設組成。
_Avoid_: 工期、承諾工時、Story Point

**過期產物（Stale Artifact）**:
因上游資料已更新而不再代表最新分析狀態的既有產物；它會保留供追溯，但不得視為目前結果。
_Avoid_: 無效資料、已刪除產物
