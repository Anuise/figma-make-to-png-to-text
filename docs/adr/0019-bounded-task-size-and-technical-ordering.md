# 限制工項大小並區分技術順序與商業優先級

自動產生的工程工項一般以基準工程師 2–16 小時可完成且可獨立驗收為目標；超過 16 小時時應依交付結果拆分，無法安全拆分時標記 `Needs Breakdown`，少於 2 小時且沒有獨立驗收價值時合併至相關工項。每筆工項預設使用可驗證 checklist 描述正常流程、錯誤狀態、權限或資料條件、UI、API／資料模型、測試範圍及未確認假設；只有具明確事件與狀態轉換的行為才使用 Given／When／Then，且每條條件都必須可追溯。工具依工項、API、資料模型與工作流相依關係產生建議實作順序，但不推測商業優先級；Excel 保留 `Business Priority`、`Owner` 與 `Sprint` 供人工填寫。
