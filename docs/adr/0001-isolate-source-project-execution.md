# 隔離執行來源專案且不轉換其框架

分析工具採用 Next.js 與 React，但來源專案維持 Figma Make 匯出時的 React 技術棧。使用者啟動前設定來源專案父目錄，Docker Compose 將它唯讀掛載至固定容器路徑，UI 只保存並選擇其下的相對子路徑。每次分析作業建立獨立工作副本；依賴安裝、暫時修改及網站執行都只發生在 Docker Compose 管理的隔離環境中，Playwright 也從獨立容器存取網站。這可避免污染主機或原始碼，並確保截圖與分析忠於原始輸入；代價是更換來源父目錄需要修改環境設定並重啟 Compose，工具也必須管理工作副本、容器間網路與來源專案的啟動契約。
